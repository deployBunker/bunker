# Package: `proto/bunker/v1/bunkerv1connect`

## Public API

- `BunkerServiceClient` — connect-go client interface for the Bunkerd service.
- `NewBunkerServiceClient(httpClient, baseURL, opts...)` — creates a gRPC/Connect/HTTP client.
- `BunkerServiceHandler` — server-side handler interface that the provider implements.
- `NewBunkerServiceHandler(svc, opts...)` — wraps a service implementation into HTTP route handlers.
- `AgentServiceClient` / `AgentServiceHandler` — same pattern for the per-agent scoped service.
- All RPCs: `SpawnAgent`, `DestroyAgent`, `ListAgents`, `GetAgent`, `ServerMetrics`, `Heartbeat`, `Exec`, `Run`, `SetEnv`.

## Conventions

- Auto-generated by `protoc-gen-connect-go`. DO NOT EDIT.
- Source: `proto/bunker/v1/bunker.proto`.
- Uses `connectrpc.com/connect` — supports gRPC, gRPC-Web, and Connect protocols on a single HTTP handler.
- Request/response types come from the parent package `proto/bunker/v1`.
- All handlers accept `context.Context` and return responses — no streaming RPCs defined.

## Test Patterns

- No tests in this package (generated code). Tests for connect-go handlers live in `internal/server/`.
- Connection-level testing is done via `net/http/httptest` in server tests.

## Pitfalls

1. **Regenerated on every `buf generate`.** Any manual changes will be overwritten. Edit the `.proto` file instead, then regenerate.
2. **Client constructors use `WithBaseURL`.** The base URL must include the scheme and host (e.g., `http://localhost:9090`). Omitting the scheme causes connection errors.
3. **Version compatibility check at init.** The `const _ = connect.IsAtLeastVersion1_17_0` assertion will fail at compile time if the connect-go version is too old.
4. **Handler wrappers are thin.** `NewBunkerServiceHandler` just delegates to the implementation's methods. All auth, logging, and middleware must be applied at the HTTP router level (chi middleware).
